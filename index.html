🚀 Getting Started

Installation

  1. Download the PrinterMake installer from the website
  2. Run the installer and follow the setup wizard
  3. Create a PrinterMake account or sign in with Google
  4. Launch the app!

Your First Game

Step 1: Create a New Project

Click the Create tab to start a new project. You'll be prompted to choose a template:

  • Starter Scene - Comes with example objects and scripts to learn from
  • Flat Baseplate - Blank canvas for building from scratch

Step 2: Explore the Editor

The Studio Editor has three main areas:

  • Left Panel - Explorer tree showing all objects in your scene
  • Center - 3D viewport where you build your game
  • Right Panel - Properties and scripting controls

Step 3: Add Objects

Use the Insert menu to add objects:

  • Box, Sphere, Cylinder (basic shapes)
  • Mirror, SpawnPoint (special objects)

Step 4: Add Interactivity

Select an object and go to the Script tab in the properties panel to add:

  • Visual scripts (event-action system)
  • C# scripts (advanced programming)

Step 5: Test Your Game

Click the Play button (▶) to test your game. Use:

  • WASD - Move around
  • Mouse Wheel - Zoom (third-person to first-person)
  • Right Click + Drag - Rotate camera (third-person only)
  • E Key - Interact with objects
  • Space - Jump

Controls Reference

Control Action
W / A / S / D Move forward / left / back / right
Space Jump
Shift Sprint (hold while moving)
Mouse Wheel Zoom camera in/out
Right Click + Drag Rotate camera (third-person)
E Interact with nearby objects

UI Overview

Main Menu Tabs:
• Home - See friends and available games

Tips for Beginners

  • Start with the Starter Scene template to see examples
  • Lock objects you don't want to accidentally move by checking the "Locked" checkbox
  • Use the right-click context menu in the Explorer to quickly delete objects
  • Test your game frequently during development

🎨 Studio Editor

The PrinterMake Studio is a full-featured 3D editor for creating games and experiences.

Editor Layout

The editor is divided into three main panels:

  • Explorer (Left) - Hierarchical tree of all objects in your scene
  • Viewport (Center) - 3D view where you build and interact with objects
  • Properties (Right) - Edit properties, appearance, scripts for selected object

Adding Objects

Use the Insert menu to add 3D objects:

Basic Shapes:
• Box - 3D rectangular shape
• Sphere - 3D round object
• Cylinder - 3D cylindrical shape

Special Objects:
• SpawnPoint - Where players appear when joining
• Mirror - Reflective surface

Properties Panel

The Properties panel lets you control:

  • Transform - Position, rotation, scale
  • Appearance - Color, material, textures
  • Physics - Collision, gravity
  • Scripts - Add interactivity

Textures

Add custom textures to objects to make them look more realistic:

  1. Select an object in the Explorer
  2. Go to Properties → Appearance → Texture
  3. Click "Upload Texture"
  4. Choose an image file (PNG, JPG, etc.)
  5. The texture will be applied to the object

Right-Click Context Menu

Right-click on objects in the Explorer for quick actions:

  • Delete - Remove the object from the scene
  • Duplicate - Create a copy
  • Rename - Change the object name

⚙️ Scripting

Make your games interactive with visual scripts or C# programming.

Visual Scripting

Visual scripting uses an event-action system that's easy to learn:

Events:
• onTouch - When the object is touched
• onPress - When clicked
• onTimer - After a time delay
• onChat - When a chat message is received
• onStart - When the game starts
Actions:
• message - Send a message
• move - Change position
• rotate - Change rotation
• toggle - Turn on/off
• destroy - Delete the object
• teleport - Move to a location
• color - Change color

C# Scripting

For advanced developers, use C# to create complex game logic:

public class MyScript : GameBehavior {
    public void Start() {
        Debug.Log("Game started!");
    }

    public void Update() {
        if (Input.GetKeyDown("w")) {
            this.transform.position += Vector3.forward;
        }
    }
}

Available APIs

C# scripts have access to these classes:

  • GameObject - Represents objects in your scene
  • Transform - Position, rotation, scale
  • Vector3 - 3D coordinates
  • Input - Keyboard and mouse input
  • Time - Frame rate and delta time
  • Debug - Logging and debugging

Creating a Script

  1. Select an object in the Explorer
  2. Go to Properties → Script tab
  3. Choose "+ Visual Script" or "⚙ C# Script"
  4. For Visual: Set the event and action
  5. For C#: Write your code in the editor
  6. Click "Save & Apply"

🎮 Publishing

Share your games with the PrinterMake community.

Before Publishing

Make sure your game is ready:

  • Test the game thoroughly in Play mode
  • Ensure there's a SpawnPoint for players
  • Add textures and polish visuals
  • Create a catchy title and description
  • Choose a thumbnail image

Publishing Steps

Step 1: Open Publish Menu

Go to File → Publish Game

Step 2: Set Game Details

Fill in:

  • Game Title
  • Description (what is your game about?)
  • Thumbnail Image

Step 3: Review & Publish

Click "Publish" to make your game live!

Managing Published Games

After publishing, you can:

  • Edit Details - Change title, description, thumbnail
  • Edit Game - Make changes to the game and re-publish
  • Delete - Remove from the community
  • View Stats - See how many players have played

Export & Import

PrinterMake uses the custom .prmk format for games:

  • Export - File → Export PrinterMake (.prmk) to save locally
  • Import - File → Import PrinterMake (.prmk) to load a saved game

🔌 API Reference

Complete API documentation for PrinterMake backend and scripting.

Game API Endpoints

GET  /api/games              - List all published games
GET  /api/games/:id          - Get game details
POST /api/games/publish      - Publish a new game
PUT  /api/games/:id/update   - Update game details
DELETE /api/games/:id/delete - Delete a game

Session API

POST /api/sessions/join           - Join a game session
POST /api/sessions/:id/leave      - Leave a session
POST /api/sessions/:id/chat       - Send chat message
GET  /api/sessions/active         - Get active sessions

User API

GET  /api/users/:id              - Get user profile
POST /api/users/friends/add       - Send friend request
POST /api/users/friends/accept    - Accept friend request
POST /api/users/friends/remove    - Remove friend
GET  /api/users/:id/friends       - List user's friends

Authentication

All API requests require a Bearer token in the Authorization header:

Authorization: Bearer YOUR_TOKEN_HERE

Tokens are obtained through Clerk authentication and are valid for 1 hour.

💡 Examples

Learn by example with these common game patterns.

Example 1: Door System

Create a door that opens when touched:

1. Create a Box for the door
2. Add Visual Script: Event = onTouch, Action = move
3. Set move target to slide door open
4. Players can now open doors by touching them!

Example 2: Collectible Items

Create items players can collect:

1. Create a Sphere for the item
2. Add Visual Script: Event = onTouch, Action = destroy
3. Also add: Action = message with "+1 Point"
4. Items disappear when collected!

Example 3: Timed Events

Create events that happen after a delay:

public class TimedEvent : GameBehavior {
    public void Start() {
        Time.delay = 3.0f; // 3 seconds
        this.DoSomething();
    }

    public void DoSomething() {
        Debug.Log("3 seconds have passed!");
        this.gameObject.SetActive(false);
    }
}

Example 4: Player Movement Script

public class PlayerController : GameBehavior {
    public float moveSpeed = 5.0f;

    public void Update() {
        Vector3 input = new Vector3(
            Input.GetAxis("horizontal"),
            0,
            Input.GetAxis("vertical")
        );

        this.transform.position += input * moveSpeed * Time.deltaTime;
    }
}
